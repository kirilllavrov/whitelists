# Whitelists — инструменты проверки доступности российских сервисов в режиме "Белых списков"

Набор Python-скриптов для автоматической проверки доступности доменов и IP-адресов российских онлайн-сервисов во время включения операторами мобильной связи режима "Белые списки". Проект предназначен для формирования whitelist'ов, чтобы исключить важные сервисы из туннелирования и обеспечить их работу через прямое соединение.

## 📋 Возможности

### check-domains.py
Проверка доменов на доступность через HTTPS:
- ✅ Эмуляция браузера на Android (Chrome Mobile)
- ✅ Асинхронная проверка с настраиваемой параллельностью
- ✅ Двухэтапная проверка: DNS + HTTP
- ✅ Fallback на httpx при ошибках протокола
- ✅ Поддержка кастомных DNS-серверов
- ✅ Классификация ошибок (DNS, SSL, Timeout, HTTP и др.)
- ✅ Выбор мобильного оператора для маркировки результатов

### check_ips.py
Сканирование IP-адресов и CIDR-диапазонов:
- ✅ Поддержка форматов: отдельные IP и CIDR-блоки
- ✅ Многопоточный ping с настраиваемым количеством потоков
- ✅ Автоматическая агрегация результатов в CIDR
- ✅ Защита от переполнения памяти (лимиты на очередь и размер сетей)
- ✅ Статистика в реальном времени

### merged.py
Консолидация результатов проверки доменов:
- ✅ Объединение whitelist'ов от разных операторов
- ✅ Фильтрация по порогу встречаемости (%)
- ✅ Добавление обязательных доменов из конфигурации
- ✅ Сортировка и удаление дубликатов

## 🐧 Установка и использование на Linux

### 1. Скачивание репозитория

```bash
# Клонируйте репозиторий
git clone https://github.com/kirilllavrov/whitelists.git
cd whitelists

# Или скачайте ZIP-архив и распакуйте
wget https://github.com/kirilllavrov/whitelists/archive/refs/heads/main.zip
unzip main.zip
cd whitelists
```

### 2. Установка зависимостей

```bash
# Создайте виртуальное окружение (рекомендуется)
python3 -m venv venv
source venv/bin/activate

# Установите зависимости для check-domains.py
pip install aiohttp httpx aiodns

# Для check_ips.py дополнительные зависимости не требуются
```

### 3. Проверка доменов

```bash
cd tools

# Базовая проверка (использует домены из ../src/domains/)
python3 check-domains.py

# Проверка с настройками
python3 check-domains.py ../src/domains \
    -c 10 \                                 # 10 параллельных запросов (по умолчанию 5)
    -e category-gov-ru \                    # Исключить категорию
    --dns 77.88.8.8 77.88.8.1 \             # Использовать кастомные DNS
    --verify-ssl                            # Включить проверку SSL

# Тихий режим (без подробного вывода)
python3 check-domains.py -q
```

**После проверки:**
- Скрипт предложит выбрать мобильного оператора (1-6)
- Результат сохранится в `../build/domains_checked/whitelist-<TIMESTAMP>-<OPERATOR>.txt`

### 4. Проверка IP-адресов

```bash
cd tools

# Сначала создайте директорию src/IPs и поместите туда файлы с IP/CIDR
mkdir -p ../src/IPs
# Поместите файлы .txt с IP или CIDR в ../src/IPs/

# Запуск проверки
python3 check_ips.py
```

**Настройки** (редактируются в начале скрипта `check_ips.py`):
- `NUM_THREADS` — количество потоков (по умолчанию 500)
- `PING_TIMEOUT` — таймаут ping в секундах (по умолчанию 3)
- `MAX_QUEUE_SIZE` — размер очереди задач (по умолчанию 10000)

**Результат:**
- Доступные IP сохраняются в `../build/IP_checked/available_ips_from_<FILENAME>.txt`
- Автоматически агрегируются в CIDR-блоки

### 5. Консолидация результатов

```bash
cd tools

# Объединение всех whitelist'ов из build/domains_checked/
python3 merged.py
```

**Настройки** (редактируются в начале скрипта `merged.py`):
- `THRESHOLD_PERCENT` — порог встречаемости (по умолчанию 75%)
- `MANDATORY_FILE` — файл с обязательными доменами

**Результат:**
- Финальный список сохраняется в `../release/whitelist.txt`

## 🪟 Установка и использование на Windows

### 1. Скачивание репозитория

**Вариант A: Через Git**
```powershell
# Установите Git для Windows: https://git-scm.com/download/win
git clone https://github.com/kirilllavrov/whitelists.git
cd whitelists
```

**Вариант B: ZIP-архив**
1. Скачайте ZIP-архив репозитория через браузер
2. Распакуйте в удобную директорию (например, `C:\whitelists`)
3. Откройте PowerShell или Command Prompt и перейдите в директорию:
   ```powershell
   cd C:\whitelists
   ```

### 2. Установка зависимостей

```powershell
# Проверьте версию Python (требуется 3.8+)
python --version

# Создайте виртуальное окружение
python -m venv venv

# Активируйте виртуальное окружение
.\venv\Scripts\Activate.ps1

# Если возникла ошибка выполнения скриптов, выполните в PowerShell от имени администратора:
# Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

# Установите зависимости
pip install aiohttp httpx aiodns
```

### 3. Проверка доменов

```powershell
cd tools

# Базовая проверка
python check-domains.py

# Проверка с настройками
python check-domains.py ..\src\domains -c 10 -e category-gov-ru --dns 8.8.8.8 1.1.1.1

# Тихий режим
python check-domains.py -q
```

**Примечание:** В Windows пути используют обратные слеши (`\`), но Python также принимает прямые (`/`).

### 4. Проверка IP-адресов

```powershell
cd tools

# Создайте директорию для IP-файлов
mkdir ..\src\IPs

# Поместите файлы .txt с IP или CIDR в ..\src\IPs\

# Запуск проверки
python check_ips.py
```

**Важно:** В Windows требуется разрешение на отправку ICMP-запросов (ping). Если ping блокируется брандмауэром:
1. Откройте «Брандмауэр Защитника Windows»
2. Разрешите «Запросы эха ICMP (ICMP Echo Request)»
3. Или запустите скрипт от имени администратора

### 5. Консолидация результатов

```powershell
cd tools
python merged.py
```

## 📊 Интерпретация результатов

### Статусы проверки доменов

|    Статус    |         Значение         |              Действие              |
|--------------|--------------------------|------------------------------------|
| ✅ OK        | Домен доступен (2xx/3xx) | Добавить в whitelist               |
| 🔐 SSL_ERR   | Ошибка SSL handshake     | Проверить вручную                  |
| 🕐 TIMEOUT   | Превышено время ожидания | Повторить позже                    |
| ❌ RST       | Соединение сброшено      | Возможно, блокировка               |
| 🌐 DNS_ERR   | Домен не резолвится      | Проверить DNS                      |
| ⚠️ HTTP_ERR  | Ошибка HTTP (4xx/5xx)    | Сервер работает, это НЕ блокировка |
| 🚫 UNREACH   | Хост недоступен          | Проверить маршрут                  |
| ❓ UNKNOWN   | Неизвестная ошибка       | Требуется анализ                   |
--------------------------------------------------------------------------------

**Входные файлы доменов** (`src/domains/`):

## ❓ FAQ

**Q: Почему некоторые домены показывают HTTP_ERR?**  
A: Это означает, что сервер ответил с ошибкой (4xx/5xx). Домен доступен, это НЕ блокировка. Такие домены можно включать в whitelist.

**Q: Как увеличить скорость проверки?**  
A: Увеличьте параметр `-c` для доменов (но не более 20-30) или `NUM_THREADS` для IP. Будьте осторожны: слишком высокие значения могут вызвать блокировку со стороны серверов.

**Q: Можно ли использовать системные DNS?**  
A: Да, не указывайте параметр `--dns`. Для принудительного использования системных DNS убедитесь, что они корректно настроены в ОС.

**Q: Где найти итоговый whitelist?**  
A: После запуска `merged.py` файл `whitelist.txt` появится в директории `release/`.

**Q: Check_ips.py не работает на Windows**  
A: Убедитесь, что брандмауэр разрешает ICMP-запросы (ping). Запустите PowerShell от имени администратора или настройте правила брандмауэра.

## 🤝 Вклад в проект

Мы стремимся создать максимально полный и точный whitelist российских сервисов, доступных через разных мобильных операторов. Поскольку доступность доменов может отличаться в зависимости от провайдера и региона, **нам важна ваша помощь!**

### Как вы можете помочь:

1. **Запустите проверку доменов** для вашего мобильного оператора (инструкции выше)
2. **Отправьте нам результат** — файл `whitelist-<TIMESTAMP>-<OPERATOR>.txt` из директории `build/domains_checked/`
3. **Создайте Issue** в репозитории с прикреплённым файлом или отправьте его любым удобным способом

### Почему это важно:

- ✅ Чем больше проверок от разных операторов — тем точнее итоговый whitelist
- ✅ Мы сможем выявить домены, доступные только у конкретных провайдеров
- ✅ Объединённые данные позволят создать универсальный список для всех пользователей
- ✅ Вы поможете тысячам людей настроить корректную работу российских сервисов

### Что мы делаем с полученными данными:

1. Собираем все файлы `whitelist-*-<OPERATOR>.txt` от сообщества
2. Еженедельно в автоматическом режиме запускаем `merged.py` для консолидации с порогом встречаемости 75%
3. Добавляем обязательные домены из конфигурации
4. Публикуем обновлённый `release/whitelist.txt` для всех пользователей

**Каждый присланный файл делает общий whitelist лучше! 🙏**

Альтернативно, вы можете:
- Предложить новые домены для проверки через Pull Request в `src/domains/`
- Сообщить об ошибках в работе скриптов через Issues
- Улучшить документацию или добавить новые функции
